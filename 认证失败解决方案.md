# 失败原因和解决方案

## 🔴 问题

你看到所有评估任务都失败了：
```
✗ Failed: Qwen/Qwen3-VL-4B-Instruct
✗ Failed: Qwen/Qwen3-VL-4B-Thinking
✗ Failed: Qwen/Qwen3-VL-8B-Instruct
...
```

**根本原因**: 缺少HuggingFace认证token，无法访问VSI-Bench数据集。

## ✅ 最快的解决方法（3步搞定）

### 第1步：获取Token

1. 访问：https://huggingface.co/settings/tokens
2. 点击 "New token" 或 "Create new token"
3. 名称随意（如：`vsi-bench`），权限选择 **"Read"**
4. 复制生成的token（格式：`hf_xxxxxxxxxxxxxx`）

### 第2步：登录HuggingFace

```bash
# 安装CLI（如果没有）
pip install huggingface_hub

# 登录
huggingface-cli login

# 粘贴刚才复制的token，按回车
```

### 第3步：重新运行

```bash
./run_parallel.sh
```

## 🎯 其他方法

### 方法2：使用环境变量（推荐用于服务器）

```bash
# 设置环境变量
export HF_TOKEN="hf_your_token_here"

# 运行评估
./run_parallel.sh
```

### 方法3：命令行参数

```bash
# 单个模型
python evaluate_vsibench.py \
    --model_name "Qwen/Qwen3-VL-4B-Instruct" \
    --gpu_id 0 \
    --hf_token "hf_your_token_here"
```

## 📊 常见问题

### Q: 我没有HuggingFace账户
**A**: 免费注册：https://huggingface.co/join

### Q: VSI-Bench需要申请访问权限吗？
**A**: 可能需要。访问 https://huggingface.co/datasets/nyu-visionx/VSI-Bench 查看是否需要点击"Request access"

### Q: Token安全吗？
**A**:
- ✅ 使用 `huggingface-cli login` 最安全（加密存储）
- ⚠️ 环境变量是明文，注意不要泄露
- ❌ 不要把token提交到Git

### Q: 我已经登录了还是失败
**A**: 验证登录状态：
```bash
huggingface-cli whoami
```
如果显示你的用户名，说明已登录。可能是数据集访问权限问题。
