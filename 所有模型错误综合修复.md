## ✅ 我的综合修复

### 修复1：Qwen视频处理（3层fallback）

```python
def infer_video(self, video_path: str, question: str, ...):
    # 确保绝对路径
    video_path = os.path.abspath(video_path)

    try:
        # 方法1: 使用qwen-vl-utils (官方推荐)
        from qwen_vl_utils import process_vision_info
        # ... process_vision_info处理视频
    except ImportError:
        # 方法2: 使用file:// URL格式
        messages = [{"type": "video", "video": f"file://{video_path}", ...}]
    except Exception:
        # 方法3: 提取帧作为图像序列
        frames = extract_frames(video_path)
        messages = [{"type": "image", "image": frame} for frame in frames]
```

**核心改进**：
1. ✅ 使用绝对路径（`os.path.abspath`）
2. ✅ 优先使用`qwen-vl-utils.process_vision_info`
3. ✅ Fallback到file://格式
4. ✅ 最后fallback到帧提取

### 修复2：Qwen模型类导入（已修复）

代码已经有正确的fallback机制：
```python
try:
    from transformers import Qwen2VLForConditionalGeneration
except ImportError:
    try:
        from transformers import AutoModelForVision2Seq
    except ImportError:
        from transformers import AutoModel
```

✅ Thinking和Instruct模型都使用同一个Qwen3VLEvaluator类，所以修复是统一的。

### 修复3：LLaVA torch_dtype兼容性

```python
try:
    # transformers >= 5.0: 使用dtype
    model = AutoModel.from_pretrained(..., dtype=torch.bfloat16)
except TypeError:
    # transformers < 5.0: 使用torch_dtype
    model = AutoModel.from_pretrained(..., torch_dtype=torch.bfloat16)
```

## 🔍 问题根源分析

### Qwen视频格式问题
**根本原因**：
- Qwen3-VL的官方推荐使用`qwen-vl-utils.process_vision_info`处理视频
- 直接使用`file://`路径可能不被某些版本支持
- 需要绝对路径而非相对路径

**解决方案**：
1. 安装`qwen-vl-utils`：`pip install qwen-vl-utils`
2. 使用`process_vision_info`处理视频输入
3. Fallback到帧提取

### LLaVA torch_dtype警告
**根本原因**：
- transformers 5.0+ 改用`dtype`参数
- 旧代码使用`torch_dtype`会触发DeprecationWarning

**解决方案**：
- Try-except处理两种参数名
- 优先使用`dtype`，失败则fallback到`torch_dtype`
